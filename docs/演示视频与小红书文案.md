补拍时只录三段：把 `.m4v` 拖入 App、微信收到文件、Premiere 导出后自动发送。先关闭通知，并换成测试文件名和测试目录，避免账号、客户素材和路径入镜。

## 小红书文案

### 标题

微信客户端发不了 m4v，我做了个 Mac 菜单栏工具

### 正文

微信客户端发不了 `.m4v`。

我之前只能打开网页版文件传输助手，扫码登录，再把文件传过去。登录一失效，又要重新扫一次。

所以我做了 WeClaw Send。文件拖进去，菜单栏里直接发。多选、进度、取消、失败重试都有。

它也不只是剪辑师用。只要你要从 Mac 往微信发 `.m4v` 就行。做后期的话，可以装 Premiere Pro 插件或 DaVinci Resolve 脚本，导出完自动发；Premiere 支持完整序列或 I/O 范围，也会记住预设和输出目录。

隐私说明：WeClaw Send 不设自有云端存储，不会向开发者上传文件或账号信息。文件仅通过微信官方 ClawBot API 发送至用户本人的微信 ClawBot；登录凭据、偏好设置和传输记录仅保存在本机。

先说清楚一个限制：WeClaw Send 用的是微信官方 ClawBot 接口。同一个微信的 ClawBot 通道，不能一边连 WeClaw Send，一边再给其它 OpenClaw / ClawBot 服务用。

目前支持 macOS 14+。因为没有购买 Apple Developer ID，第一次打开需要在“隐私与安全性”里手动允许。

项目已开源，GitHub 搜索 `WeClawSend`。
