### 真实操作补拍

补拍时只录四段：把 `.m4v` 拖入 App、微信收到文件、Premiere 导出后自动发送、设置中切换到 OpenClaw 登录。先关闭通知，并换成测试文件名、测试目录和测试账号，避免真实账号、客户素材和路径入镜。

## 小红书文案

### 标题

微信客户端发不了 m4v，我做了个菜单栏发送工具

### 正文

微信客户端发不了 `.m4v`。

我之前只能打开网页版文件传输助手，扫码登录，再把文件传过去。登录一失效，又要重新扫一次。

所以我做了 WeClaw Send。文件拖进去，菜单栏里直接发。多选、进度、取消、失败重试都有。

它不只给剪辑师用。只要需要从 Mac 往微信发 `.m4v`，拖进去就行。做后期的话，还支持 Premiere Pro、Final Cut Pro 和 DaVinci Resolve，导出完成后可以自动发送。

如果 OpenClaw 已经登录了微信，不需要重新扫码。在设置中把登录方式切到 OpenClaw，WeClaw Send 会只读使用当前账号和会话。它不会复制或修改 OpenClaw 的登录文件，也不会收取微信消息；发送时保持 OpenClaw 运行即可。多个微信账号会要求手动选择。

隐私说明：WeClaw Send 不设自有云端存储，不会向开发者上传文件或账号信息。文件仅通过微信官方 ClawBot API 发送至用户本人的微信 ClawBot；登录凭据、偏好设置和传输记录仅保存在本机。

目前支持 macOS 14+。因为没有购买 Apple Developer ID，第一次打开需要在“隐私与安全性”里手动允许。

项目已开源，GitHub 搜索 `WeClawSend`。
